# 📌 Overview

**Dead Shot**は、制限時間内に迫り来るゾンビを撃破し、高スコアを目指す3Dサバイバルアクションゲームです。

本作品は、Unityを用いたゲーム開発を通して、ゲームシステム設計・オブジェクト指向設計・UI/UX設計への理解を深めることを目的として制作しました。

企画から設計、プログラミング、UI制作、ゲームバランス調整まで、すべて個人で担当しています。

# 🏗 System Architecture

ゲーム全体を役割ごとに分割し、それぞれのクラスが単一の責務を持つことを意識して設計しました。

```text
                      Player
                         │
        ┌────────────────┼────────────────┐
        │                │                │
        ▼                ▼                ▼
 WeaponAmmoSystem   EnemyManager   TutorialUIManager
        │                │
        ▼                ▼
 EnemyAttack      SpawnManager
        │                │
        └────────┬───────┘
                 ▼
          Game Management
      (Time・Score・Audio)
                 │
                 ▼
          Result Management
```

## 主なクラス

|クラス|役割|
|:--|:--|
|WeaponAmmoSystem|武器・残弾・リロード管理|
|EnemyManager|敵生成・敵管理|
|EnemyAttack|敵AI・攻撃処理|
|SpawnManager|敵スポーン管理|
|TimeKeeper|制限時間管理|
|TutorialUIManager|チュートリアル管理|
|UIGameController|ゲームUI管理|
|AudioManager|BGM・効果音管理|
|GameResultData|ゲーム結果保存|
|ResultScoreDisplay|リザルト画面表示|

各システムを独立して設計することで、機能追加や仕様変更の影響範囲を最小限に抑え、保守性・拡張性を高めています。

# 💡 Challenges & Solutions

## ① 戦闘システムの構築

本作品で最も苦労したのは、プレイヤー・敵・UI・ゲーム進行が連携する戦闘システムの構築です。

戦闘では単純に敵へダメージを与えるだけではなく、

- 武器・残弾管理
- 命中判定
- 敵HP管理
- 撃破判定
- スコア加算
- 撃破数更新
- UI更新
- リザルトデータ保存

など、多数のシステムがリアルタイムで連携する必要がありました。

開発当初は各処理の依存関係が強く、仕様変更のたびに他システムへ影響が及ぶことが課題でした。

そこで、武器管理・敵管理・ゲーム管理・UI管理を役割ごとに分離し、責務を明確にしたことで、保守性・拡張性を向上させました。

---

## ② プレイヤーアニメーションの構築

プレイヤーの操作に対して自然なアニメーションを実現することにも苦労しました。

待機・歩行・走行・射撃・リロードなど複数のアニメーションをAnimatorで管理していましたが、

- 入力タイミングによるアニメーションの競合
- 射撃中に移動した際の不自然な遷移
- リロード完了前の状態遷移

などが発生し、操作感を損ねる場面がありました。

そこでAnimator Controllerのステート構成や遷移条件を見直し、パラメータ管理を整理することで、プレイヤーの入力とアニメーションが自然に同期するよう改善しました。

また、プレイテストを繰り返しながら細かな調整を行い、違和感の少ない操作感を実現しました。

---

## ③ 戦闘UIの構築

戦闘中はプレイヤーへ多くの情報を伝える必要があり、

- HP
- 残弾数
- スコア
- 撃破数
- 制限時間
- ミニマップ

をリアルタイムで表示しています。

一方で、情報量を増やし過ぎると視認性が低下し、ゲームプレイの妨げとなるため、FPSゲームを参考にしながら情報の優先順位を整理しました。

重要な情報ほど視線移動の少ない位置へ配置し、HPバーのアニメーションやリザルト画面を含めたデザインを統一することで、プレイヤーが直感的に状況を把握できるUIを目指しました。

---

## ④ プレイヤー体験を意識した改善

ゲーム制作では「機能を実装すること」だけでなく、「最後まで楽しく遊べること」を重視しました。

そのため、プレイテストを繰り返し実施し、

- 敵のスポーン数
- 攻撃頻度
- 武器性能
- スコア配分
- UI表示
- 演出

などを継続的に改善しました。

また、ゲーム開始時にはヘリコプターからの降下演出やチュートリアルを導入し、初めてプレイするユーザーでも自然にゲームへ入り込める構成としています。
