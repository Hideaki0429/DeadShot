# 📌 Overview

Dead Shotは、制限時間内に迫り来るゾンビを撃破し、高スコアを目指す3Dサバイバルアクションゲームです。

本作品は、Unityを用いたゲーム開発を通して、ゲームシステム設計やオブジェクト指向設計、UI/UX設計の理解を深めることを目的として制作しました。

企画・設計・プログラミング・UI制作・ゲームバランス調整まで、すべて個人で担当しています。

# 🏗 System Architecture

ゲーム全体を役割ごとに分割し、それぞれのクラスが単一の責務を持つことを意識して設計しました。

                      Player
                         │
        ┌────────────────┼────────────────┐
        │                │                │
        ▼                ▼                ▼
 WeaponAmmo      EnemyManager      TutorialUI
    │                │
    ▼                ▼
 EnemyAttack      Enemy AI
        │
        ▼
 Game Management
(Time・Score・Audio)
        │
        ▼
  Result Management
```

## 主なクラス

|クラス|役割|
|:--|:--|
|WeaponAmmoSystem|武器・残弾・リロード管理|
|EnemyManager|敵生成・敵管理|
|EnemyAttack|敵AI・攻撃処理|
|SpawnManager|敵スポーン管理|
|TimeKeeper|ゲーム時間管理|
|TutorialUIManager|チュートリアル管理|
|UIGameController|ゲームUI管理|
|AudioManager|BGM・効果音管理|
|GameResultData|ゲーム結果保存|
|ResultScoreDisplay|リザルト画面表示|

各システムを独立して設計することで、機能追加や仕様変更を行う際の影響範囲を最小限に抑え、保守性・拡張性を高めています。

# 💡 Challenges & Solutions

## ① 戦闘システムの構築

最も苦労したのは、プレイヤー・敵・UI・ゲーム進行が連携する戦闘システムの構築です。

戦闘では射撃処理だけでなく、残弾管理、命中判定、敵HP更新、撃破判定、スコア加算、撃破数更新、UI更新、リザルトデータ保存など、多くの処理が同時に動作します。

開発当初は各処理が密結合になっており、一部の仕様変更が他の機能へ影響することが課題でした。

そこで、武器管理・敵管理・ゲーム管理・UI管理をそれぞれ独立したクラスへ分割し、責務を明確化しました。これにより、保守性を向上させるとともに、新しい武器や敵を追加しやすい設計を実現しました。

---

## ② プレイヤーアニメーションの構築

プレイヤーの操作とアニメーションを自然に同期させることにも苦労しました。

待機・歩行・走行・射撃・リロードなど複数のアニメーションをAnimatorで制御していましたが、入力タイミングによってアニメーションが途中で中断されたり、不自然な遷移が発生する問題がありました。

Animatorのステート構成や遷移条件を見直し、各状態の切り替えを整理することで、プレイヤーの入力に対して違和感のない滑らかなアニメーション遷移を実現しました。

また、プレイテストを繰り返しながら細かな調整を行い、操作感の向上にも取り組みました。

---

## ③ 戦闘UIの構築

戦闘中は、HP・残弾数・スコア・撃破数・制限時間・ミニマップなど、多くの情報をリアルタイムで表示する必要があります。

情報量が多くなると視認性が低下し、ゲームプレイに支障をきたすため、FPSゲームを参考に情報の優先順位を整理し、重要な情報ほど自然に視認できるレイアウトを設計しました。

また、HPゲージのアニメーションやチュートリアル画面、リザルト画面までデザインを統一し、一貫したゲーム体験となるよう工夫しました。

---

## ④ プレイヤー体験を意識した改善

本作品では「機能を実装すること」だけでなく、「最後まで楽しく遊べること」を重視しました。

そのため、敵のスポーン数や攻撃頻度、武器性能、スコア配分、UI表示、演出などについて何度もプレイテストを実施し、改善を繰り返しました。

特にゲーム開始時のヘリコプター降下演出からチュートリアル、ゲーム開始までの流れは、初めてプレイするユーザーでも自然にゲームへ入り込めるよう細かな調整を行っています。
